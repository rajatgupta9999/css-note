CSS

  • Cascading Style Sheet
  • It is a language that is used to describe the style of a document written in HTML

Types of CSS

  1. Inline CSS

    • Styles directly within the HTML element using the style attribute
    • Used to apply styles to individual elements
    • Pros:
      • High specificity (overrides other styles).
      • Useful for testing or quick fixes.
    • Cons:
      • Hard to maintain and reuse.
      • Mixes content with presentation (not recommended for large projects).
  2. Internal CSS
    • Styles are defined within a style tag in the head section of an HTML document.
    • Use Case: Styling a single HTML page.
    • Pros:
      • Keeps styles separate from HTML content..
      • Useful for small projects or single-page websites.
    • Cons:
      • Not reusable across multiple pages.
      • Increases page load time for larger styles
  3. External CSS
    • Styles are written in a separate .css file and linked to the HTML document using the link tag.
    • Use Case: Styling multiple pages in a website.
    • Pros:
      • Reusable across multiple pages.
      • Improves maintainability and separation of concerns.
      • Reduces page load time (cached by the browser).
    • Cons:
      • Requires an additional HTTP request to load the CSS file.

Selectors

  1. Universal Selector:-
    - These are the most commonly used selectors.

    1. Element Selector :-
      2. - Selects all elements of the specified type.
      ex:- p{Color:Blue;}
      .This selects all < p> element and make their color blue
      .
    2. Class Selector :-
      3. - Selects all elements with the specified class.
      ex:- .class{Color:Blue;}
      .This selects all elements with the class name "class" and make their color blue
      .
    3. Id Selector :-
      4. - Selects a single element with the specified id attribute
      . ex:- #id{Color:Blue;}
      .This selects the element with the id "id" and make their color blue
      .
    4. Universal Selector :-
      5. - Selects all elements on the page.
      ex:- *{Color:Blue;}
      .This selects all elements on the page and make their color blue
      .

  2. Combinators:-
    - These are used to select elements based on their relationship to other elements.
    - Combinators allows you to combine multiple selectors to target specific relationship between elements.

    1. Descendant Selector :-
      2. - Selects all elements that are descendants of the specified element.
      ex:- div p{Color:Blue;}
      .This selects all < p> elements that are descendants of < div> elements and make their color blue
      .
    2. Child Selector :-
      3. - Selects all elements that are direct children of the specified element.
      ex:- ui > li {Color:Blue;}
      .This selects all < li> elements that are direct children of < ui> elements and make their color blue
      .
    3. Adjacent Sibling Selector :-
      4. - selects an element that is immediately preceded by the specified element.
      - Selects all elements that are adjacent siblings of the specified element.
      ex:- h1 + p {Color:Blue;}
      .This selects all < p> elements that comes immediately after an < h1>
      .
    4. General Sibling Selector :-
      5. - Selects all elements that are siblings of the specified element.
      ex:- p ~ p {Color:Blue;}
      .This selects all < p> elements that are siblings of < p> elements and make their color blue

  3. Attribute selector:-
    - These are used to select elements based on their relationship to other elements.
    - These selectors target element based on their attributes

    1. Attribute Presence :-
      2. - Selects a elements with the specified attribute.
      ex:- a[target] {Color:Blue;}
      .This selects all < a> elements with a "target" attribute and make their color blue
      .
    2. Attribute Value :-
      3. - Selects a elements with the specified attribute value.
      ex:- a[target=_blank] {Color:Blue;}
      .This selects all < a> elements with a "target" attribute value "_blank" and make their color blue
      .
    3. Attribute contains :-
      4. Selects element whose attribute value contains the specified substring .
      ex:- a[href*="google"] {Color:Blue;}
      .This selects all < a> elements whose attribute value contains "google" and make their color blue
      .
    4. Attribute Starts With :-
      5. Selects element whose attribute value starts with the specified value.
      ex:- a[href^="https://"] {Color:Blue;}
      .This selects all < a> elements whose attribute value starts with "https://" and make their color blue
      .
    5. Attribute Ends With :-
      6. Selects element whose attribute value ends with the specified value.
      ex:- a[href$=".com"] {Color:Blue;}
      .This selects all < a> elements whose attribute value ends with ".com" and make their color blue

  4. Pseudo classes:-
    - Pseudo classes are are used to target specific elements based on their position within a parent container.

    1. Hover :-
      2. - Selects the element when it is hovered over.
      ex:- a:hover {Color:Blue;}
      .This selects all < a> elements that are hovered over and make their color blue
      .
    2. Focus :-
      3. - Selects the element when it is focused(eg, via keyboard navigation).

      .
    3. First child :-
      4. - Selects the first child element of the parent element.
      ex:- ul li:first-child {Color:Blue;}
      .This selects all < li> elements that are first child of < ul> elements and make their color blue
      .
    4. Last child :-
      5. - Selects the last child element of the parent element.
      ex:- ul li:last-child {Color:Blue;}
      .This selects all < li> elements that are last child of < ul> elements and make their color blue

      6. - Selects the x child element of the parent element.

Flex Box

• Flexbox (Flexible Box Layout) is a CSS layout module designed to create flexible and responsive layouts. It allows you to easily align and distribute space among items in a container, even when their sizes are unknown or dynamic.

The Flex model

• Flex model is a great way to get more flexibility in your layouts and to simplify responsive layout design. It makes it easy to align elements on a 2D plane and is pretty easy to use once you get familiar with the main properties.
• The first step is to set display: flex on a container element. The children to the flex container become flex items. A set of properties can be applied to flex containers, and have an effect to all the items as a whole, and a different set of properties can be applied to flex items and have their effect on the targeted items. Flex items can in turn also be flex containers for the elements it contains, making it easy to create complex layouts.

- The Main Axis and Cross Axis is the axis along which the flex items are placed in the flex container and it is changeable according to the content direction.
  • Properties and Values:-
    1. Flexbox Direction:-
      2. -It sets how flex items are placed in the flex container, along which axis and direction.
      • Values:-
        1. row:-
          2. -Flex items are placed horizontally, from left to right.
          -The main axis runs horizontally (left to right).
        2. row-reverse:-
          3. - Flex items are placed horizontally, but in reverse order (right to left).
          -The main axis runs horizontally (right to left).
        3. column:-
          4. -Flex items are placed vertically, from top to bottom.
          -The main axis runs vertically (top to bottom).
        4. column-reverse:-
          5. -Flex items are placed vertically, but in reverse order (bottom to top).
          -The main axis runs vertically (bottom to top).
    2. align-items:-
      3. - align-items property is used to align flex items along the cross axis of the flex container. The cross axis is perpendicular to the main axis (which is defined by the flex-direction property). The align-items property ensures that all flex items are aligned consistently within the container.
      • Values:-
        1. stretch (default):-
          2. -Flex items are stretched to fill the container along the cross axis.
        2. flex-start:-
          3. - Flex items are aligned to the start of the cross axis.
        3. flex-end:-
          4. -Flex items are aligned to the end of the cross axis.
        4. center:-
          5. -Flex items are centered along the cross axis.
        5. baseline:-
          6. -Flex items are aligned such that their baselines align.
          -This is useful when dealing with text or items of varying heights.
    3. justify-content:-
      4. - the justify-content property is used to align flex items along the main axis of the flex container.
      - The justify-content property controls how space is distributed between and around flex items.
      • Values:-
        1. flex-start (default):-
          2. -Flex items are packed toward the start of the main axis.
        2. flex-end:-
          3. - Flex items are packed toward the end of the main axis.
        3. center:-
          4. -Flex items are centered along the main axis.
        4. space-between:-
          5. -Flex items are evenly distributed along the main axis, with the first item at the start and the last item at the end
        5. space-around:-
          6. -Flex items are evenly distributed along the main axis, with equal space around them. last item at the end
        6. space-evenly:-
          7. -Flex items are evenly distributed along the main axis, with equal space between and around them.

          2. -All flex items are placed in a single line, even if they overflow the container.

        3. wrap-reverse:-
          4. -Flex items wrap onto multiple lines, but in reverse order (bottom to top or right to left).
    5. flex-grow:-
      6. - the flex-grow property controls how much a flex item will grow relative to other flex items in the same container when there is extra space available. It determines the ability of a flex item to expand and take up remaining space along the main axis.
      • Values:-
        • All flex items are placed in a single line, even if they overflow the container.
        • It defines the proportion of the available space that the item should take up.
        • If all items have flex-grow: 1, they will share the available space equally.
        • If one item has flex-grow: 2 and the others have flex-grow: 1, the first item will take up twice as much space as the others.
    6. flex-shrink:-
      7. - the flex-shrink property controls how much a flex item will shrink relative to other flex items in the same container when there is not enough space available. It determines the ability of a flex item to reduce its size along the main axis.
      • Values:-
        • The flex-shrink property accepts a unitless value (a positive number, default is 1).
        • It defines the proportion of the negative space (overflow) that the item should shrink.
        • If all items have flex-shrink: 1, they will shrink equally.
        • If one item has flex-shrink: 2 and the others have flex-shrink: 1, the first item will shrink twice as much as the others.
    7. align-content:-
      8. - the align-content property is used to align multiple lines of flex items along the cross axis when there is extra space in the container. This property only works when the flex container has multiple lines (i.e., when flex-wrap is set to wrap or wrap-reverse).
      • Values:-
        1. stretch (default):-

          5. -Lines of flex items are centered along the cross axis.

          6. -Lines of flex items are evenly distributed along the cross axis, with the first line at the start and the last line at the end.
        6. space-around:-
          7. -Lines of flex items are evenly distributed along the cross axis, with equal space around them.
        7. space-evenly:-
          8. -Lines of flex items are evenly distributed along the cross axis, with equal space between and around them.
    8. align-self:-
      9. - the align-self property is used to align an individual flex item along the cross axis of the flex container. It overrides the align-items value for that specific item, allowing you to customize the alignment of individual items within the container.
      • Values:-
        1. auto (default):-

          3. - The item is aligned to the start of the cross axis.

          4. -The item is aligned to the end of the cross axis.

Grid

Grid is a powerful layout system that allows you to create complex, responsive designs with ease. It works by dividing a container into rows and columns, enabling precise control over the placement and sizing of items.
  • Key Concepts
    1. Grid Container:-
      • The parent element that holds the grid items.
      • Created by setting display: grid; or display: inline-grid;
    2. Grid Items:-
      • The child elements inside the grid container.
    3. Grid Lines:-
      • The horizontal and vertical lines that divide the grid into rows and columns.
      • Lines are numbered starting from 1.
    4. Grid Tracks:-
      • The space between two grid lines (rows or columns).
    5. Grid Cells:-
      • The smallest unit in the grid, formed by the intersection of a row and a column.
    6. Grid Areas:-
      • A rectangular area formed by combining multiple grid cells.
  1. Grid Container Properties
    1. display: grid:-
      • Defines the container as a grid.
    2. grid-template-columns and grid-template-rows:-
      • Defines the size of columns and rows.
      • Mostly use Fractional Unit (fr):- Distributes available space proportionally.
    3. grid-template-areas:-
      • Defines named grid areas.
    4. gap:-
      • Adds space between grid items.
    5. justify-content and align-content:-
      • Aligns the entire grid within the container.
    6. justify-items and align-items:-
      • Aligns items within their grid cells.
  2. Grid Item Properties
    1. grid-column and grid-row:-
      • Places items in specific columns or rows.
    2. grid-area:-
      • Places items in named grid areas.
    3. justify-self and align-self:-
      • Aligns an individual item within its grid cell.

Media Queries

• Media Queries in CSS are used to apply different styles based on the characteristics of the device or viewport, such as screen width, height, orientation, and resolution. They are essential for creating responsive designs that adapt to various devices like desktops, tablets, and mobile phones.
• Media quearies have different targets
• Logical Operators: Use logical operators like and, or, and not to combine multiple media queries.
  • Targets:-
    • all(defalut):- Applies to all devices.
    • screen:- Applies to screens (e.g., desktops, tablets, phones, laptops).
    • print:-Applies to printed documents or print previews.
      •Useful for creating print-friendly styles.
    • speech:- Applies to screen readers and other speech synthesis devices.
      •Used for accessibility purposes.
    • aural:- Intended for speech synthesizers (replaced by speech).
    • braille:- Intended for braille tactile feedback devices.
    • embossed:- Intended for paged braille printers.
    • handheld:- Intended for handheld devices (e.g., smartphones).
    • projection:- Intended for projected presentations.
    • tty:- Intended for teletypes and terminals.
    • tv:- Intended for television-type devices.
  • Values:-

      • 1. Viewport/Device Dimensions

    • width:- Exact viewport width (e.g., width: 768px).
    • height:- Exact viewport height.
    • min-width:-Minimum viewport width (e.g., min-width: 768px).
    • max-width:- Maximum viewport width (e.g., max-width: 1024px).
    • min-height:- Minimum viewport height.
    • max-height:- Maximum viewport height.

      • 2. Orientation

    • orientation:- Checks if the device is in portrait or landscape mode.

      • 3. Resolution

    • resolution:- Checks the pixel density of the device (e.g., min-resolution: 2dppx for high-DPI screens).

      • 4. Aspect Ratio

    • aspect-ratio:- Ratio of viewport width to height (e.g., aspect-ratio: 16/9).
    • min-aspect-ratio:- Minimum aspect ratio.
    • max-aspect-ratio:- Maximum aspect ratio.

      • 5. Interaction Capabilities

    • hover:- Checks if the device supports hover (e.g., hover: hover for mice, hover: none for touchscreens).
    • pointer:- Checks the primary input mechanism (fine for mice, coarse for touchscreens).

      • 6. Color and Display

    • color:- Checks the number of bits per color component (e.g., min-color: 8).
    • color-gamut:- Checks the color space (e.g., color-gamut: p3 for wide-gamut displays).
    • monochrome:- Checks if the device is monochrome.

      • 7. Accessibility Preferences

    • prefers-color-scheme:- Detects dark/light mode (e.g., prefers-color-scheme: dark).
    • prefers-reduced-motion:- Checks if the user prefers reduced motion.

      • 8. Scripting Support

    • scripting:- Checks if JavaScript is enabled (scripting: enabled or scripting: none).

Transitions

CSS Transitions allow you to smoothly animate changes to CSS properties over a specified duration. They create fluid visual effects (e.g., hover effects, color changes, or element movements) without JavaScript.
  • Value:-
  1. transition-property:-
    2. Specifies which CSS properties to animate.
    - Example: transition-property: background-color, width;
    -The transition-property property can take:
    • all: Applies transitions to all properties
    • Specific property names (e.g., width, background-color etc)
    • A comma-separated list of properties
    • none: Disables transitions
  2. transition-duration:-
    3. Defines how long the transition lasts (e.g., 1s, 200ms).
    - Example:transition-duration: 0.3s;
  3. transition-timing-function:-
    4. The transition-timing-function property in CSS specifies how the intermediate values of a CSS property being transitioned are calculated over the duration of the transition. Essentially, it defines the speed curve of the transition effect, allowing you to control the acceleration and deceleration.
    • value:-
      1. ease (default):-
        2. The transition starts slowly, accelerates in the middle, and then slows down at the end.
      2. linear:-
        3. The transition occurs at a constant speed from start to end.
      3. ease-in:-
        4. The transition starts slowly and accelerates as it progresses.
      4. ease-out:-
        5. The transition starts quickly and decelerates as it progresses.
      5. ease-in-out:-
        6. The transition starts slowly, accelerates in the middle, and then decelerates at the end.
      6. cubic-bezier(n,n,n,n):-
        7. Allows you to define a custom speed curve using a cubic Bézier function. The parameters are four numbers between 0 and 1 that define the control points of the curve.
      7. steps(n, start|end):-
        8. The transition is divided into a specified number of steps, with the change occurring at the start or end of each step.
  4. transition-delay:-
    5. Adds a delay before the transition starts.
    - Example:transition-delay: 0.2s;

Transition Shorthand

transition: [property] [duration] [timing-function] [delay];
- Example:transition: font-size 2s ease-in-out 0.2s;

CSS Transform

CSS Transforms allow you to modify the shape, size, position, or orientation of elements in 2D or 3D space. They are widely used for animations, hover effects, and creating dynamic layouts without affecting the document flow.
  • Value:-
  1. rotate:-
    2. It is used to rotate an element around a fixed point (by default, its center) in 2D or 3D space. which allows you to apply visual transformations to elements.
    • Degrees (deg): [e.g., 45deg (45 degrees clockwise)].
    • Radians (rad): [e.g., 1.57rad (≈90 degrees)].
    • Turns (turn): [e.g., 0.25turn (90 degrees)].
    • Gradians (grad):[e.g., 100grad (90 degrees)].
    1. Basic Rotation:-
      2. Rotate an element clockwise or counterclockwise
      Ex:- transform: rotate(30deg); Rotate 30° clockwise
    2. Rotation Origin:-
      3. By default, rotation occurs around the element’s center. Use transform-origin to change the pivot point
      Ex:-transform-origin: top left; transform: rotate(45deg); Rotate around the top-left corner of the element.
    3. 3D Rotation:-
      4. For 3D effects, combine with other rotation functions
      1. rotateX():-
        2. Rotate around the X-axis (vertical flip).
      2. rotateY():-
        3. Rotate around the Y-axis (horizontal flip).
      3. rotateZ():-
        4. Same as rotate(default Z-axis).
  2. scale:-
    3. It is used to resize an element by scaling it up or down. It modifies the size of an element without affecting its layout in the document flow, making it ideal for animations, hover effects, and visual transformations.
    • sx: Scaling factor along the X-axis (horizontal).
      1 = original size, 2 = double size, 0.5 = half size.
    • sy: (Optional) Scaling factor along the Y-axis (vertical).If omitted, sy defaults to sx.
    1. Uniform Scaling:-
      2. Scale an element equally in both directions.
      Ex:- transform: scale(1.5); Scales to 1.5x the original size.
    2. Non-Uniform Scaling:-
      3. Scale an element differently along the X and Y axes.
      Ex:-transform: scale(2, 0.5); Doubles width, halves height.
    3. Scaling Origin:-
      4. By default, scaling occurs from the element’s center. Use transform-origin to change the pivot point.
      Ex:-transform-origin: top left; transform: scale(1.5); Scales from the top-left corner.
  3. translate:-
    4. It is used to move an element from its current position along the X-axis (horizontal), Y-axis (vertical), or both. Unlike positioning with top, left, etc., translate() does not affect the document flow, making it ideal for animations and visual adjustments.
    • tx: Distance to move along the X-axis (horizontal).
      -Positive values move the element to the right, negative values to the left.
    • ty: (Optional) Distance to move along the Y-axis (vertical).If omitted, ty defaults to 0.
      -Positive values move the element down, negative values up.
    1. Basic Translation:-
      2. Move an element horizontally and/or vertically
      Ex:- transform: translate(50px, 20px); Move 50px right, 20px down
    2. Single-Axis Translation:-
      3. Use translateX() or translateY() for single-axis movement
      Ex:-[transform: translateX(50px); Move 50px right] [transform: translateY(-20px); Move 20px up ]
  4. skew:-
    5. It is part of the transform property and is used to distort an element by tilting it along the X-axis (horizontal) and/or Y-axis (vertical). This creates a "skewed" or slanted effect, which can be useful for creative designs and animations.
    • ax: Angle of skew along the X-axis (horizontal).
      - positive values tilt the element to the left, negative values to the right.
    • ay: (Optional) Angle of skew along the Y-axis (vertical).If omitted,
      -Positive values move the element down, negative values up.
      - ay defaults to 0.
    1. Basic Skew:-
      2. Skew an element along the X-axis and/or Y-axis
      Ex:- transform: skew(20deg, 10deg); Skew 20° horizontally, 10° vertically
    2. Single-Axis Skew:-
      3. Use skewX() or skewY() for single-axis skewing
      Ex:-[transform: skewX(20deg); Skew 20° horizontally] [transform: skewY(-10deg); Skew 10° vertically (upward)
  5. matrix:-
    6. It is a 2D transformation function that allows you to apply a variety of transformations to an element, all within a single function. It's a powerful tool, though it can be a bit complex to understand at first.
    • The matrix() function defines a 2D transformation matrix. This matrix essentially modifies the coordinates of an element, resulting in transformations like:
      • Translation (moving the element)
      • Scaling (resizing the element)
      • Rotation (turning the element)
      • Skewing (distorting the element)
    • Syntax:-
      • - The matrix() function takes six values representing a 2D transformation matrix:
      Ex [transform: matrix(a, b, c, d, e, f);]
      • a Horizontal scaling (scaleX).
      • b Vertical skewing (skewY).
      • c Horizontal skewing (skewX).
      • d Vertical scaling (scaleY).
      • e Horizontal translation (translateX).
      • f Vertical translation (translateY).

Animation

CSS Animations allow you to create smooth, complex animations by defining keyframes and applying them to elements. Unlike transitions, animations can have multiple intermediate steps and can run automatically or be triggered by events.
  1. @keyframes:-
    Firstly we have to write @keyframes and then write the name of the animation. and then we have to write the animation star from to end.[Basically we have to write the animation Frames start frame or end frame].
    • Syntax:- @keyframe myName {
      to { initial property : initial value ; }
      from { end property : end frame ; }
      }
    • Ex:- @keyframe animate font {
      to { font-size : 20px; }
      from { font-size : 40px; }
      }
  2. percentage(%) in @keyframes animations:-
    Percentages are used to define the intermediate steps (or keyframes) of an animation. Each percentage represents a specific point in time during the animation's duration, allowing you to control the element's state at that moment.
    Syntax:- @keyframes animation-name {
    0% { /* Starting state */ }
    50% { /* Intermediate state */ }
    100% { /* Ending state */ }
    }
  3. Steps :-
    1. animation name:-
      2. -Specifies the name of the @keyframes animation to use.
      -It links the element to the animation defined in the @keyframes rule.
      Syntax:- [animation-name: name of @keyframe;]
    2. animation Duration:-
      3. Sets the length of time the animation takes to complete (e.g., 2s, 500ms).
      Ex:- [animation-duration: 1s;]
    3. animation-timing-function
      4. Controls the speed curve of the animation (e.g., ease, linear, ease-in-out).
      Ex:- [animation-timing-function: ease;]
    4. animation-delay:-
      5. Adds a delay before the animation starts.
      Ex:- [animation-delay: 1s;]
    5. animation-iteration-count:-
      6. Specifies how many times the animation should play (e.g., infinite).
      Ex:- [animation-iteration-count: infinite;]
    6. animation-Direction:-
      7. Controls the direction of the animation.
      1. normal (default):-
        2. The animation plays forward from 0% to 100%
      2. reverse:-
        3. The animation plays backward from 100% to 0%
      3. alternate:-
        4. The animation alternates between forward and backward on each iteration.
      4. alternate-reverse:-
        5. The animation alternates between backward and forward on each iteration.
      syntax:- [animation-direction: normal | reverse | alternate | alternate-reverse;]
    7. animation-fill-mode:-
      8. Determines the styles applied before and after the animation (e.g., forwards, backwards,both).
      Ex:- [animation-fill-mode: none;]
    8. animation-play-state:-
      9. Pauses or resumes the animation (e.g., paused, running).
      Ex:- [animation-play-state: value;]

    Animation Shorthand

    animation: [name] [duration] [timing-function] [delay] [iteration-count] [direction] [fill-mode] [play-state];
    - Example:transition: font-size 2s ease-in-out 0.2s;
