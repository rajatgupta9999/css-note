CSS

  • Cascading Style Sheet
  • It is a language that is used to describe the style of a document written in HTML

Types of CSS

  1. Inline CSS

    • Styles directly within the HTML element using the style attribute
    • Used to apply styles to individual elements
    • Pros:
      • High specificity (overrides other styles).
      • Useful for testing or quick fixes.
    • Cons:
      • Hard to maintain and reuse.
      • Mixes content with presentation (not recommended for large projects).
  2. Internal CSS
    • Styles are defined within a style tag in the head section of an HTML document.
    • Use Case: Styling a single HTML page.
    • Pros:
      • Keeps styles separate from HTML content..
      • Useful for small projects or single-page websites.
    • Cons:
      • Not reusable across multiple pages.
      • Increases page load time for larger styles
  3. External CSS
    • Styles are written in a separate .css file and linked to the HTML document using the link tag.
    • Use Case: Styling multiple pages in a website.
    • Pros:
      • Reusable across multiple pages.
      • Improves maintainability and separation of concerns.
      • Reduces page load time (cached by the browser).
    • Cons:
      • Requires an additional HTTP request to load the CSS file.

Selectors

  1. Universal Selector:-
    - These are the most commonly used selectors.

    1. Element Selector :-
      2. - Selects all elements of the specified type.
      ex:- p{Color:Blue;}
      .This selects all < p> element and make their color blue
      .
    2. Class Selector :-
      3. - Selects all elements with the specified class.
      ex:- .class{Color:Blue;}
      .This selects all elements with the class name "class" and make their color blue
      .
    3. Id Selector :-
      4. - Selects a single element with the specified id attribute
      . ex:- #id{Color:Blue;}
      .This selects the element with the id "id" and make their color blue
      .
    4. Universal Selector :-
      5. - Selects all elements on the page.
      ex:- *{Color:Blue;}
      .This selects all elements on the page and make their color blue
      .

  2. Combinators:-
    - These are used to select elements based on their relationship to other elements.
    - Combinators allows you to combine multiple selectors to target specific relationship between elements.

    1. Descendant Selector :-
      2. - Selects all elements that are descendants of the specified element.
      ex:- div p{Color:Blue;}
      .This selects all < p> elements that are descendants of < div> elements and make their color blue
      .
    2. Child Selector :-
      3. - Selects all elements that are direct children of the specified element.
      ex:- ui > li {Color:Blue;}
      .This selects all < li> elements that are direct children of < ui> elements and make their color blue
      .
    3. Adjacent Sibling Selector :-
      4. - selects an element that is immediately preceded by the specified element.
      - Selects all elements that are adjacent siblings of the specified element.
      ex:- h1 + p {Color:Blue;}
      .This selects all < p> elements that comes immediately after an < h1>
      .
    4. General Sibling Selector :-
      5. - Selects all elements that are siblings of the specified element.
      ex:- p ~ p {Color:Blue;}
      .This selects all < p> elements that are siblings of < p> elements and make their color blue

  3. Attribute selector:-
    - These are used to select elements based on their relationship to other elements.
    - These selectors target element based on their attributes

    1. Attribute Presence :-
      2. - Selects a elements with the specified attribute.
      ex:- a[target] {Color:Blue;}
      .This selects all < a> elements with a "target" attribute and make their color blue
      .
    2. Attribute Value :-
      3. - Selects a elements with the specified attribute value.
      ex:- a[target=_blank] {Color:Blue;}
      .This selects all < a> elements with a "target" attribute value "_blank" and make their color blue
      .
    3. Attribute contains :-
      4. Selects element whose attribute value contains the specified substring .
      ex:- a[href*="google"] {Color:Blue;}
      .This selects all < a> elements whose attribute value contains "google" and make their color blue
      .
    4. Attribute Starts With :-
      5. Selects element whose attribute value starts with the specified value.
      ex:- a[href^="https://"] {Color:Blue;}
      .This selects all < a> elements whose attribute value starts with "https://" and make their color blue
      .
    5. Attribute Ends With :-
      6. Selects element whose attribute value ends with the specified value.
      ex:- a[href$=".com"] {Color:Blue;}
      .This selects all < a> elements whose attribute value ends with ".com" and make their color blue

  4. Pseudo classes:-
    - Pseudo classes are are used to target specific elements based on their position within a parent container.

    1. Hover :-
      2. - Selects the element when it is hovered over.
      ex:- a:hover {Color:Blue;}
      .This selects all < a> elements that are hovered over and make their color blue
      .
    2. Focus :-
      3. - Selects the element when it is focused(eg, via keyboard navigation).

      .
    3. First child :-
      4. - Selects the first child element of the parent element.
      ex:- ul li:first-child {Color:Blue;}
      .This selects all < li> elements that are first child of < ul> elements and make their color blue
      .
    4. Last child :-
      5. - Selects the last child element of the parent element.
      ex:- ul li:last-child {Color:Blue;}
      .This selects all < li> elements that are last child of < ul> elements and make their color blue

      6. - Selects the x child element of the parent element.

    26. -The text-underline-offset CSS property controls the distance (offset) between a text underline and the text itself. It allows you to adjust the position of the underline, making it closer to or farther from the text. This property is particularly useful for fine-tuning the appearance of underlined text, especially when working with custom fonts or complex layouts.
    • Values:-
      1. auto (default)
  26. text-underline-position:-
    27. -The text-underline-position CSS property controls the position of underlines applied to text using text-decoration: underline. It allows you to adjust where the underline is drawn relative to the text baseline, which is especially useful for improving readability in languages with complex glyphs or vertical writing modes.
    • Values:-
      1. auto (default)::-
        2. - The browser automatically positions the underline. This usually places it slightly below the baseline, which may overlap with descenders (e.g., in letters like "g" or "y").
      2. from-font:-
        3. - Uses metrics from the font file (if available) to determine the underline position. If the font lacks this data, it falls back to auto.
      3. under:-
        4. - Forces the underline to be drawn below the baseline, avoiding clashes with descenders. This is ideal for readability in horizontal text.
      4. left / right:-
        5. - Positions the underline to the left or right of the text, used in vertical writing modes (common in East Asian languages like Japanese or Chinese).
      5. Combinations::-
        6. - under left / under right: Combines under with a directional offset for vertical text.
  27. text-wrap:-
    28. -The text-wrap CSS property controls how text is wrapped within its container. It determines whether text should break lines to fit within the available space or avoid wrapping altogether. This property is particularly useful for improving readability and managing text layout in responsive designs.
    • Values:-
      1. wrap (default)::-
        2. - Text breaks into multiple lines to fit within the container's width.
      2. nowrap:-
        3. - Text does not break into multiple lines. It stays on a single line, potentially overflowing the container.
      3. balance:-
        4. - Attempts to balance the text across multiple lines, making each line roughly the same length. This is useful for headings or short blocks of text to create a more visually appealing layout.
      4. pretty:-
        5. - Similar to wrap, but optimizes line breaks to improve readability and avoid awkward breaks (e.g., hyphenation or orphaned words). This is experimental and may not be widely supported.
  28. text-wrap-mode:-
    29. -The text-wrap-mode CSS property is part of the CSS Text Module Level 4 and is used to control how text wraps within its container. It determines whether text should wrap to the next line when it reaches the edge of its container or if it should overflow instead. This property is particularly useful for managing text layout in responsive designs and ensuring readability.
    • Values:-
      1. wrap (default)::-
        2. - Text breaks into multiple lines to fit within the container's width.
      2. nowrap:-
        3. - Text does not break into multiple lines. It stays on a single line, potentially overflowing the container.
  29. text-wrap-style:-
    30. -The text-wrap-style CSS property is part of the CSS Text Module Level 4 (still in draft as of 2023) and is designed to control how text is wrapped and formatted within its container. However, this property is not yet standardized and has very limited browser support. It is part of a broader effort to improve text layout control in CSS.
    • Values:-
      1. auto (default)::-
        2. - The browser handles line breaks automatically, using its default algorithm.
      2. balance:-
        3. - Attempts to balance text across lines to create visually even line lengths (similar to text-wrap: balance).
      3. pretty:-
        4. - Optimizes line breaks to avoid awkward hyphenation and orphaned words, improving readability.
      4. stable:-
        5. - Prioritizes consistent line breaks when content changes dynamically (prevents jarring reflows).
      5. < custom-ident>:-
        6. - Experimental value for future extensions (not yet defined).

Font

In CSS, font-related properties are used to control the appearance of text, including its typeface, size, weight, style, and more. Below is a comprehensive guide to the most commonly used font properties:

  1. font-family:-
    2. -Specifies the typeface for text.
    -The font-family property lists one or more font families, separated by commas. Each font family is specified as either a < family-name> or a < generic-name> value.
    • Values:-
      1. < family-name>:-
        2. - The name of a font family.
        - use one or more family names separated by commas.
        - For example, "Arial", "Times New Roman", etc.
      2. < generic-name>:-
        3. -These are broad categories that browsers use as fallbacks:
        -This have mainy 5 fonts family
        -serif, sans-serif, monospace, cursive, fantasy.
  2. font-size:-
    3. -Sets the size of the text
    • Values:-
      1. Absolute units:-
        2. px, pt, in, cm, mm
      2. Relative units:-
        3. em, rem, %, vh, vw
  3. font-weight:-
    4. -Controls the thickness of the text.
    • Values:-
      1. Keywords:-
        2. normal, bold, bolder, lighter
      2. Numeric:-
        3. 100, 200, ..., 900 (400 = normal, 700 = bold)
  4. font-stylet:-
    5. -Sets the style of the text (e.g., italic).
    • Values:-
      1. normal:-
        2. Default style
      2. italic:-

    6. -Controls small-caps or normal text.

  6. line-height:-
    7. -Sets the height of a line of text.

        3. px, em, rem
  7. font (Shorthand Property):-
    8. -Combines multiple font properties into one declaration.
  8. @font-face (Custom Fonts):-
    9. -Allows you to use custom fonts not installed on the user's system.
  9. font-variant-emoji:-
    10. -The font-variant-emoji property in CSS is used to control the rendering of emoji characters in text. -It allows you to specify how emojis should be displayed, such as using a text-style or emoji-style representation.
    • Values:-
      1. normal (default)::-
        2. Emojis are displayed in their default style (usually colorful and graphical).
